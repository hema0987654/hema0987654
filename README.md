<h2>🧩 System Design & Engineering Philosophy</h2>

<div style="border-left: 5px solid #0EA5E9; background: #F8FAFC; border-radius:12px; padding:1.8rem;">
  <p style="font-size:1.3em; font-weight:700; color:#0F172A;">“Design for failure, optimize for clarity.”</p>
  <p style="color:#334155; line-height:1.7;">
    Every system I build starts with the domain –  
    understanding the real business problem before writing a single line of code.  
    I translate complexity into decoupled services, clean interfaces,  
    and pragmatic trade-offs.  
    <br><br>
    I favour simplicity over cleverness, and I measure success by  
    maintainability, fault tolerance, and how gracefully the system  
    degrades when things go wrong.  
    The next developer who reads my code should feel like they‘ve found  
    a well‑organised handbook, not a puzzle.
  </p>
  <p style="color:#0EA5E9; font-weight:600;">
    Core principles: Loose coupling · High cohesion · Statelessness ·  
    Async‑first thinking · Observability by design
  </p>
</div>
